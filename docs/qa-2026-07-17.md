## 발견 이슈와 조치 (전부 수정 완료)

1. **P1 — 사용자 SnG '봇 채우기' 영구 거부** (agent-5): create-room이 모든 SnG를 wallet 모드로
   강제하는데 wallet SnG는 휴먼 6명 에스크로 전용이라 `sng-fill-bots`가 도달 불가능한 죽은 경로였다
   (UI/문서와 모순). → SnG 참가 방식 `economyMode`('wallet' 기본 | 'practice') 추가. 연습 SnG는
   무료·봇 채우기 가능, 지갑 SnG는 버튼 숨김+사유 안내. 커밋 `ffbbf28`, 5R로 실서버 재검증.
2. **P1 — 폴드 승리 시 승자 홀카드 노출** (agent-10): endHand가 폴드 승리에도 street='showdown'을
   세팅하는데 revealed 판정이 경합 여부를 구분하지 않아 승자 실카드가 상대에게 노출됐다 (표준 룰
   위반, 기존부터 있던 버그). → getPublicState가 쇼다운 '경합'(생존자 ≥2)일 때만 공개하도록 수정
   + 회귀 테스트, 실서버 스모크로 머킹 확인.
3. **P3 — room-lost 문구가 원인 무관하게 '서버가 재시작되어…' 고정** (agent-7): grace 만료·타 탭
   퇴장도 같은 문구였다. → 중립 문구('게임 세션이 만료되어…')로 교체.
4. **P2 — create-room bigBlind 상한 없음** (agent-8): 1e15 허용 시 min/maxBuyIn이 안전 정수 초과.
   → [2, 1000] 클램프 + 회귀 테스트.
5. **P1(개발환경) — Node 23.8 미만에서 dev 서버 기동 불가** (QA 준비 중 발견): 시작 백업이
   node:sqlite native backup을 요구. → dev는 미지원 시 경고 후 스킵, 프로덕션은 기존대로 하드 실패.
   커밋 `912020b`.
