# 플레이어 좌석 폴드 시각 상태 수정 설계

## 목적

토너먼트와 캐시 게임에서 액션 결과와 캐릭터 시각 상태가 일치하도록 한다.

- 시간 초과로 자동 체크된 플레이어는 실제로 폴드하지 않았으므로 캐릭터를 흐리거나 흑백으로 만들지 않는다.
- 실제 폴드한 플레이어의 캐릭터는 현재보다 명확한 흑백·감쇠 상태로 표시한다.
- 본인이 접은 홀카드는 수트 색상은 유지하되, 폴드가 분명히 느껴질 정도로 강하게 흐린다.

## 원인

`PlayerSeat`가 `player.sitOutNext`를 모두 자리비움으로 해석한다. 시간 초과 시 서버는
`sitOutNext=true`와 `sitOutAuto=true`를 함께 설정한 뒤 체크가 가능하면 자동 체크한다.
따라서 상태는 여전히 `active`인데도 UI가 자리비움용 `grayscale`을 적용한다.

반대로 실제 폴드한 본인 좌석은 홀카드 수트 색상을 보존하려는 기존 예외 때문에 캐릭터까지
`opacity-60`만 적용되어 폴드 여부가 충분히 구분되지 않는다.

## 시각 상태 계약

좌석 표현은 다음 우선순위로 판정한다.

1. `status === 'folded'`: 실제 폴드
2. 칩이 0 이하이고 올인이 아님: 탈락
3. `status === 'sitting-out'` 또는 명시적 `sitOutNext`: 자리비움
4. 그 외: 정상

`sitOutAuto=true`인 동안의 `sitOutNext`는 같은 핸드 안에서 자리비움 시각 상태로 간주하지 않는다.
다음 핸드에 서버가 실제 자리비움 상태로 전환하면 그때 자리비움 감쇠를 적용한다.

## 렌더링

- 캐릭터 레이어
  - 실제 폴드: 강한 흑백과 낮은 불투명도
  - 자리비움·탈락: 기존 수준의 감쇠
  - 자동 체크: 정상 색상과 불투명도
- 홀카드 레이어
  - 본인 실제 폴드: `grayscale`은 사용하지 않고 불투명도를 크게 낮춘다.
  - 다른 정상·공개 카드 동작은 변경하지 않는다.
- 이름·칩 플레이트
  - 읽을 수 있도록 기존보다 약한 감쇠만 유지한다.

캐릭터와 홀카드가 같은 상위 컨테이너의 필터를 공유하지 않도록 시각 클래스를 분리한다.

## 테스트

순수 상태 판정 함수를 분리하고 다음 회귀를 테스트한다.

1. `active + sitOutNext + sitOutAuto`는 정상 표시다.
2. `folded`는 강한 폴드 캐릭터 표시다.
3. 본인 폴드 홀카드는 색상 유지용 비흑백 클래스와 강한 투명도 클래스를 사용한다.
4. 명시적 자리비움과 탈락은 계속 감쇠된다.

서버 액션 규칙은 변경하지 않으며 클라이언트 표현만 수정한다.
